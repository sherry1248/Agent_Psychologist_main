# 웹 데모 실행 방법

이 데모는 `python -m demo.app`로 실행하는 Gradio 웹 화면입니다. 외부 API 키 없이도 `LLM_TYPE=MOCK`로 동작합니다.

## 설치 명령

프로젝트 루트에서 필요한 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

Gradio가 별도로 필요하면 다음을 추가로 실행합니다.

```bash
pip install gradio
```

## 환경 변수 설정

MOCK 모드로 실행하려면 아래처럼 설정합니다.

Windows PowerShell:

```powershell
$env:LLM_TYPE = "MOCK"
```

Windows cmd:

```cmd
set LLM_TYPE=MOCK
```

## 웹 데모 실행 명령

프로젝트 루트에서 실행합니다.

```bash
python -m demo.app
```

## 브라우저 접속 주소

실행 후 브라우저에서 다음 주소로 접속합니다.

```text
http://127.0.0.1:7860
```

> 참고: 서버 로그에 `0.0.0.0`로 표시되더라도 로컬 브라우저에서는 `127.0.0.1:7860`으로 접속하세요.

## 웰니스 체크인

슬라이더는 선택 입력입니다. 잘 모르겠으면 기본값(보통)으로 두어도 괜찮습니다.

## 빠른 예시

- 일반 예시: `요즘 너무 외롭고 잠을 잘 못 자요.`
- 불안 예시: `사람 만나는 게 부담스럽고 계속 불안해요.`
- 위험 예시: `죽고 싶어요.` (위기 안내가 최우선으로 표시되어야 합니다)

## 일반 입력 테스트 예시

메시지:

```text
요즘 너무 외롭고 잠을 잘 못 자요.
```

기대 결과:

- AI 응답이 출력됩니다.
- `risk_stage`가 보입니다.
- `counseling_hint`, `empathy_style_hint`, `wellness_hint`가 출력됩니다.
- 웰니스 점수를 입력하면 `support_hint`가 함께 반영됩니다.

## 위험 입력 테스트 예시

메시지:

```text
죽고 싶어요
```

또는:

```text
자해하고 싶어요
```

기대 결과:

- 일반 상담 응답보다 109, 119, 112 안내가 먼저 출력됩니다.
- `requires_crisis_response`가 사실로 처리됩니다.
- 상담/공감/웰니스 힌트보다 위기 안내가 우선합니다.

## 오류 발생 시 확인할 것

1. `LLM_TYPE=MOCK`가 설정되어 있는지 확인합니다.
2. `gradio`가 설치되어 있는지 확인합니다.
3. 프로젝트 루트에서 실행했는지 확인합니다.
4. `http://127.0.0.1:7860` 주소를 잘못 입력하지 않았는지 확인합니다.
5. 포트 7860이 다른 프로세스에 의해 점유되지 않았는지 확인합니다.
